# lv-doctrine (source of truth)

## hierarchy
layered_ventures (umbrella)
- lvcc: layered ventures command center (management shell)
- puremetalprints: fabrication + storefront systems
- layered-studios: content production tooling + pipelines
- mechanick: diagnostic intelligence platform

## operating principles
- local-first by default; cloud integrations are optional layers
- everything important becomes a file-based artifact (md/json) that can be passed forward
- deterministic systems > vibes
- security + auditability by design (especially for mechanick / policy systems)

## logging/ops standard
- lv_ops is the shared operational brain
- all projects write to the same ops contracts over time
- carry-forward capsule is the primary continuity mechanism

## definition of "done"
- functional first, cleanup later (but never break determinism or workflow rules)
