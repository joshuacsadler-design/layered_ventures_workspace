# naming (global)

Non-negotiable naming convention for ALL projects and artifacts:

- linux-safe, cross-platform
- lowercase only
- no spaces
- allowed characters: a-z 0-9 hyphen (-) underscore (_)
- stable ids where relevant (do not rename casually)
- prefer: <domain>_<thing> or <domain>-<thing> consistently per folder
- avoid special characters, emojis, and mixedCase

Examples:
- good: project-brief.md, context-capsule.md, lv_chat.ps1, mechanick_api
- bad: Project Brief.md, LV Ops, MechaNick(API).ps1
